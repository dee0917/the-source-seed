# 🌌 Chronos Nexus: The Temporal Sovereignty Engine

## 1. 核心宗旨 (The Core Mission)
iHaveTime 的宗旨是「分析空閒」，但 **Chronos Nexus** 的目標是「主權重建」。
這不再是一個工具，而是一個**時間代理系統**。它不只是告訴你有沒有時間，而是主動為你**奪回**被熵值（Entropy）吞噬的時間。

## 2. 與 iHaveTime 的本質區別
- **iHaveTime**: 被動、分析型、日曆視角。
- **Chronos Nexus**: 主動、戰略型、意識流視角。

## 3. 三大核心機制 (Core Mechanics)
1. **Entropy Scavenger (熵值清道夫)**:
   自動掃描並消除無效的「偽空閒」。如果一個 15 分鐘的空隙不足以產生任何價值，它會主動建議將其合併或轉化為「強制休息」，而非標註為「可用」。
2. **Impact-to-Energy Mapping (影響力能量映射)**:
   不以「忙碌」為標準，而以「能量」為核心。系統會學習主人的認知高峰期，主動在最高能量時段「封鎖」外界干擾，而非等待主人自行安排。
3. **Ghost Scheduling (影子排班)**:
   在正式日曆之外，存在一個「戰略影子日曆」。代理會在這裡模擬各種時間分配方案，只有最優解才會被推送到主人的真實視野中。

## 4. 技術架構 (The Architecture)
- **Deep Engine**: 使用 Pydantic AI 與 LangGraph 構建多級決策鏈。
- **Interface**: 捨棄傳統 Dashboard，優先採用 **Telegram 戰略控制台**。
- **Memory**: 接入本地 Episodic Memory，讓系統記得「上一次你在這個時段工作時的效率極低」。
